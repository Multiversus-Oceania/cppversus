post /accounts/recover

post /accounts/recover

Kicks off the account recovery process

Triggers the account recovery process for users who forget their login credentials. Once an email address is sent to this endpoint, a new recovery token will be generated, and sent off to the email address from our servers. This token can then be passed to the /auth endpoint to recover the account, set a new password, and generate a new auth_token. Please note that the user must have added an email address to his/her account before using this endpoint.

Request BodyDescription
emailThe email address that is associated with this account.
Secured by
  • Anonymous client authentication.

Auth Tokens

post /auth

post /auth

Generates an auth token from Hydra.

An auth token takes the place of a service token or a username/password combination for obtaining a Hydra access token. Auth tokens do not have defined expiry times, but they may be revoked. If a previously-obtained auth token is revoked, the client should retrieve a new one via this endpoint.

Request BodyDescription
username[Optional] Used only for our built-in username/password authentication. Accepts any valid string.
password[Optional] Used only for our built-in username/password authentication. Accepts any valid string.
facebook[Optional] Accepts and validates a Facebook auth token for login through Facebook. If no Hydra account exists for this Facebook authentication, an account will automatically be created using this authentication method.
steam[Optional] Accepts and validates a Steam auth token for login through Steam. If no Hydra account exists for this Steam authentication, an account will automatically be created using this authentication method.
live[Optional] Accepts and validates a Windows Live ID auth token for login through Windows Live. If no Hydra account exists for this Windows Live authentication, an account will automatically be created using this authentication method.
google[Optional] Accepts and validates a Google auth token for login through Google. If no Hydra account exists for this Google authentication, an account will automatically be created using this authentication method.
uuid[Optional] Accepts a valid UUID for login. If no Hydra account exists with this UUID, an account will automatically be created using this authentication method.
recovery[Optional] Accepts a new token for login that is generated upon using our account recovery system.
twitter[Optional] Accepts and validates a Twitter access token key and access token secret for login through Twitter. If no Hydra account exists for this Twitter authentication, an account will automatically be created using this authentication method.
anonymous[Optional] Accepts true to create an anonymous account. The only way to auth after this will be with the auth token, until another authentication is linked.
custom[Optional] Accepts a custom auth token for login through Hydra's Custom Authentication feature. If no Hydra account exists for this custom authentication, an account will automatically be created using this authentication method.
shortcode[Optional] Accepts a shortcode generated by a logged-in account.
external_hydra[Optional] Accepts an External Hydra object (see below) for authenticating against a different Hydra environment
External Hydra BodyDescription
environment_slugThe slug of the configured external environment
authThe authentication data to send to the external environment
Secured by
  • Anonymous client authentication.
